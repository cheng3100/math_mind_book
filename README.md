# Math Mind Book

这个仓库用于记录和维护一些介绍“数学思想”的书籍与阅读笔记。

这里关注的不是单纯的公式、刷题或考试技巧，而是数学背后的思考方式：如何建模、如何逼近、如何抽象、如何证明、如何从粗糙版本逐步迭代到高质量结果。

## 起点：从牛顿迭代法理解数学思想

最近了解到牛顿迭代法计算平方根时，一个很重要的启发是：

> 先做出一个粗糙的 1.0 版本，然后不断迭代修正，往往比一开始就追求完美答案更好、更快。

这背后对应一种非常通用的思想：

- 不一定要一步得到最终答案；
- 可以先构造一个近似解；
- 通过反馈不断减少误差；
- 只要方向正确，过程可以快速收敛；
- 很多工程问题、算法问题、系统 debug 问题，本质上也可以用这种方式理解。

牛顿迭代法的价值不只是“求平方根”，而是展示了一种强大的求解范式：

> 猜测 → 反馈 → 修正 → 收敛。

这种思想也广泛出现在数值计算、优化算法、机器学习、控制系统、编译器优化、工程 bring-up 和复杂系统 debug 中。

## 推荐书籍

### 1. How to Solve It

作者：George Pólya

这本书是关于“如何解决问题”的经典著作。

它真正讨论的不是某个具体数学分支，而是数学家和工程师在面对未知问题时可以采用的思考方式，例如：

- 如何理解问题；
- 如何拆解问题；
- 如何先解决一个简化版本；
- 如何从特殊情况发现规律；
- 如何提出猜想；
- 如何验证和修正猜想。

这本书非常适合作为数学思想类阅读的第一本书。

它强调的很多方法，例如“先尝试一个简单情况”“从相似问题中寻找类比”“先猜再证明”，和牛顿迭代法背后的思想很接近：不要指望一开始就得到完美答案，而是通过合理的步骤逐步逼近。

### 2. Mathematics and Plausible Reasoning

作者：George Pólya

这本书可以看作是 How to Solve It 的进一步展开。

它重点讨论数学中的“合情推理”：数学发现并不总是从严格证明开始，很多时候是从观察、类比、归纳和猜测开始。

这本书特别适合用来理解：

- 数学家如何产生猜想；
- 为什么例子很重要；
- 类比在数学发现中有什么作用；
- 归纳和证明之间是什么关系；
- 为什么严格证明之前，常常先有直觉和模式识别。

它能帮助读者把数学从“定义—定理—证明”的静态结构，理解成“观察—猜想—验证—修正—抽象”的动态过程。

### 3. Concrete Mathematics

作者：Ronald Graham、Donald Knuth、Oren Patashnik

这本书非常适合计算机背景的读者。

它连接了数学与计算机科学，讨论很多在算法、程序设计和系统分析中常见的数学工具。

### 4. Numerical Recipes

作者：William H. Press 等

适合理解数学如何在计算机中落地。

### 5. Gödel, Escher, Bach

作者：Douglas Hofstadter

适合理解递归、自指、形式系统和复杂结构。
